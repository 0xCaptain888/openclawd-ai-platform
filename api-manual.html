SECTION 01

快速开始

三步完成部署，立即开始使用 AI 平台全部功能

1

复制环境配置文件

bash
cp .env.example .env
2

编辑 LLM 接口地址和密钥

.env
# 大语言模型接口（兼容 OpenAI 格式） LLM_API_URL=http://your-llm-server:8080/v1 LLM_API_KEY=sk-your-api-key-here LLM_MODEL=gpt-4o # API 网关密钥（客户端使用此 token 访问） GATEWAY_API_KEY=your-gateway-secret-token # 服务端口 GATEWAY_PORT=80
3

一键启动所有服务

bash
docker compose up -d [+] Running 5/5 ✓ Network ai-platform_default Created ✓ Container ai-gateway Started ✓ Container ai-social-bot Started ✓ Container ai-website Started ✓ Container ai-trading Started # 验证所有服务运行正常 curl http://localhost:80/health {"status":"ok","services":{"social":"healthy","website":"healthy","trading":"healthy"}}
⚡
提示：所有 API 请求统一通过网关（端口 80）访问，无需直接连接各服务端口。请求头需携带 Authorization: Bearer <GATEWAY_API_KEY>

SECTION 02

社媒运营机器人 API

一套接口覆盖小红书、抖音、微博、微信公众号、B站等主流平台的内容生成与改写

POST /social/generate

根据指定平台、话题和风格，生成一篇符合平台调性的原创内容。支持小红书、抖音、微博、公众号、B站等平台。

请求示例

curl
curl -X POST http://localhost:80/social/generate \ -H "Content-Type: application/json" \ -H "Authorization: Bearer your-gateway-secret-token" \ -d '{ "platform": "xiaohongshu", "topic": "夏日防晒好物推荐", "style": "种草", "tone": "活泼可爱", "keywords": ["防晒霜", "SPF50", "清爽不油腻"], "word_count": 500 }'

SECTION 05

API 网关

统一鉴权、限流、路由与审计日志

鉴权方式

所有 API 请求必须在请求头中携带 Bearer Token 进行身份认证：

HTTP Header
Authorization: Bearer your-gateway-secret-token

Token 在 .env 文件中通过 GATEWAY_API_KEY 配置。支持配置多个 Token（逗号分隔），分配给不同客户端：

# 多个客户端 Token（逗号分隔） GATEWAY_API_KEY=token-for-frontend,token-for-mobile,token-for-partner
⚠
注意：未携带 Token 或 Token 无效的请求将返回 401 Unauthorized。请勿将 Token 暴露在前端代码中，建议通过后端代理调用。

限流策略

网关内置令牌桶限流算法，按 Token 维度独立计数，防止 API 滥用：

限流维度默认值说明
每分钟请求数60 次/分钟单个 Token 每分钟最大请求次数
每日请求总量10,000 次/天单个 Token 每日最大请求总量
并发连接数10 个单个 Token 同时进行的请求数上限
请求体大小1 MB单个请求的 Body 最大尺寸

超出限流的请求将收到以下响应：

429 Too Many Requests
{ "success": false, "error": { "code": "RATE_LIMIT_EXCEEDED", "message": "请求频率超出限制，请稍后重试", "retry_after": 30 } }

响应头中会包含 X-RateLimit-Remaining 和 X-RateLimit-Reset 字段，方便客户端实现退避策略。

// 指数退避重试示例 async function callWithRetry(url, options, maxRetries = 3) { for (let i = 0; i < maxRetries; i++) { const res = await fetch(url, options); if (res.status !== 429) return res; const wait = Math.pow(2, i) * 1000; // 1s, 2s, 4s await new Promise(r => setTimeout(r, wait)); } throw new Error('请求超出重试次数限制'); }

# 1. 在指标目录下创建新指标文件 vim data/trading/indicators/custom_indicator.json # 2. 定义指标计算规则 { "id": "VWAP", "name": "成交量加权平均价", "description": "Volume Weighted Average Price", "params": { "period": { "default": 20, "min": 5, "max": 200 } }, "formula": "sum(close * volume) / sum(volume)", "signal_rules": { "bullish": "price > VWAP", "bearish": "price < VWAP" } } # 3. 重启 trading 服务 docker compose restart trading # 4. 在 API 调用中使用新指标 curl -X POST http://localhost:80/trading/api/indicators \ -H "Authorization: Bearer your-gateway-secret-token" \ -H "Content-Type: application/json" \ -d '{"symbol":"BTC/USDT","indicators":["VWAP"]}'
